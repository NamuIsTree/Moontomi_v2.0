MoonTomi 2.0 Dev Blueprint

차기 MoonTomi 웹 서비스를 위한 개발 청사진

웹 구축

  • Tech Stack

    React.js (SPA) + Tomcat (WAS) + Sprint Boot (Web Server)

  • URIs

    • admin (관리자 화면)

      • /admin
        Admin.js : Admin 기능 지원
        Admin 기능 : 정기 비평글 작성, 데이터 추가/삭제/갱신

    • general (일반 화면)

      • /home
        Home.js : 명예의 전당, 검색 기능 지원
        검색기능은 우선순위가 높지 않음
        명예의 전당 (최근 N개월 명예의 전당으로 수정하는 건 어떨까...)
      • /regular/{id}
        /regular/* : 정기 음평회 메인 페이지 (latest 음평회)
        Regular.js : 정기 음평회
        현재 방식을 유지할 수 있으면 좋겠지만, SEO를 고려하여 리모델링 할 예정
      • /review
        Review.js : 비평 메인 페이지
        Main Page를 통해 single tab과 album tab을 한방에 보여주면서도 복잡하지 않게
        • /single/{id}
          Single.js : 싱글 트랙 비평
          UI 고려해야함
        • /album/{id}
          Album.js : 앨범 총평
          현재 스타일을 유지하되, 필터링 기능은 수정해야함
      • /lookup
        Lookup.js : 둘러보기 (정기 음평회용)
        둘러보기 : 전체 / Season별 / 장르별 순위 (순위 지정 방식 고려중)
        작은 화면에서 더 가시적으로 볼 수 있는 방법을 찾아야해...

    • Components

      • Header
        HeaderMain.js : 홈 화면 Header (for Homepage)
        HeaderSub.js : 서브 화면 Header (for other pages)
        Pitchfork / HIPHOPLE style을 따라해볼까 생각중

      • Footer
        Footer.js : Footer
        '문형규의 토요일 : 음악에 미치는 날'
        제작자, 운영자를 지금과 다른 방식으로 보여줄 수 있을까...

API 구축

  • Tech Stack (미정)

    Node.js Express
    GraphQL + Apollo

DB 구축

  • Tech Stack

    MariaDB

  • Tables (or Collections)

    • Artists

      artist_id : artist id
      artist : 실제 이름 (영어, 한국어, 한자 … 등등)
      pron : 한국식 발음
      nation : 국적 (한국, 일본, 미국, 프랑스 … )

    • Albums

      album_id : id
      artist_id : artist id
      album_name : 앨범명
      release : 발매일 (timestamp)

    • Album_gen

      album_id : album id
      genre : 장르

    • Genres

      genre : 장르

    • Tracks

      album_id : album id
      track_num : 트랙 번호

    • Single Reviews

      single_id : single id
      writer : 작성자 닉네임 … 설정 안하면 랜덤 (익명의 ~~)
      date : 작성일
      genre : 장르 (여기는 하나만)
      youtube_url : 유투브 URL
      rating : 0 ~ 5 (step : 0.5)
      comment : 한줄평
      review : 비평글
      highlight : 자신이 꼽는 하이라이트 구간

    • Reviews

      review_id : review id
      writer : 비평 기고가 닉네임
      date : 작성일
      comment : 한줄평
      review : 비평글
      rating : 0 ~ 5 (step : 0.5)
      youtube_url : 유투브 URL
      release : 발매일

    • Regulars

      regular_id : regular id
      album_id : album id
      season : 음평회 특집번호

    • Seasons

      season : 특집 이름 (2020 상반기 특집, 쿠소 특집, Moon’s 100선 … 등등)
      season_icon : 특집 아이콘 url

    • Comments

      comment_id : comment id
      regular_id : regular id
      writer : 작성자 닉네임
      password : 비밀번호 (수정하고 싶을 때 사용)
      rating : 0 ~ 5 (step: 0.5)
      best1 : best 1 트랙 번호
      best2 : best 2 트랙 번호
      best3 : best 3 트랙 번호
